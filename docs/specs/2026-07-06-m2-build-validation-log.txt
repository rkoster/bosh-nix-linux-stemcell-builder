BUILD ATTEMPT #1 & #2: Full os-image Build
=============================================================================

Command:
  cd /home/ruben/workspace/rfc-nix-based-linuxstemcell-builder
  timeout 900 nix build ./poc#os-image 2>&1

Derivation Chain:
  • ubuntu-24.04-noble-amd64 (base image, requires runInLinuxVM)
  • os-overlay-users
  • os-overlay-ssh
  • os-overlay-sysctl-limits-env
  • os-overlay-sudoers-pam
  • os-overlay-rsyslog
  • os-overlay-audit
  • os-overlay-misc-os
  • os-overlay-systemd-services

Result: ✗ FAILED
Error: "Virtual machine didn't produce an exit code."
Nix guidance: "build failure may have been caused by lack of free disk space"

Root Cause Analysis:
  The Nix runInLinuxVM environment (used for privileged filesystem ops in
  vmTools.makeImageFromDebDist) allocates a limited internal disk to the guest VM.
  Despite 570GB free on the host, the 100GB allocated to the guest QEMU VM
  was exhausted during the debootstrap + APT package unpack phase.

  This is a known limitation of Nix's makeImageFromDebDist approach documented
  in the ubuntu-images.html reference article.
